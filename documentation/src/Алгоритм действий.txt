1). Как получить svg для интеграции в Android-приложение.

    1). Заходим на сайт https://mosmetro.ru/metro-map

    2). Нажать Ctrl + Shift + I, в открывшемся окне перейти в раздел Console.
    Ввести команду: document.getElementsByTagName('svg')[4].

    3). Найденный элемент надо скопировать. Нажать на него правой кнопкой мыши -> Copy -> Copy element

    4). Открыть скопированный код в Intellij и иправить все ошибки, которые выдает IDE:
    Код вида <image transform="matrix(1 0 0 1 0 0)"></image> исправить в <image transform="matrix(1 0 0 1 0 0)"/>.
    Аналогично с path, stop, circle, rect. В результате получаем код вообще без ворнингов.

    5). Сохранить отредактированный файл с расширением .xml. Открыть файл в браузере.
    Крта не отобразится из-за ошибки типа "error on line 205 at column 36: Entity 'nbsp' not defined".
    Проблема: некорректно закодированная спецсимвол-последовательность &nbsp;.
    Как исправить: использовать UTF-8 код неразрывного пробела, т.е. заменить "&nbsp;" "\u00A0"

    Теперь файл .xml корректно открывается. 

2). Как получить из svg скомпилированное js приложение для интеграции в android приложение
(без зума, скролла и обработки нажатий, просто чтобы в android приложении отображалась карта)

    1). В Intellij создаем новый проект Vue.js. 
    Все параметры выставлены по умолчанию, кроме названия и расположение проекта ничего не указывать.

    2). Дальнейшие инструкции по сборке Vue-приложения в один inlined html-файл
    лежат в chatgpt (ivanz8.dev) -> GatherRound -> SVG в Vue.
    
    3). Простейшая интеграция полученного index.html лежит в том же чате из предудыщего пункта.

3). Теперь надо добавит возможность масштабирования и скроллинга карты, а также обработку нажатий на станции.
Для удобства разобьем код на файлы: в Template.html лежит htmp разметка, в Script.js лежит скрипт, обрабатывающий нажатия,
в Styl.css лежат стили (?), в MatroMap.vue происхдит объединение всех трех перечисленных файлов воедино.

    1). Добавим обработку нажатий. Нужно чтобы при нажатии на кружок станции или на ее название происходила обработка события.
    Пскольку доступа к редактированию SVG у меня нет, будем прописывать оба вида нажатий (на кружок и на текст) в script. 
    С этим опять же спраляется gpt.

    Нюанс: чтобы скролл, зум и обработка нажатий заработала, необходимо убрать "pointer-events: none" в html файле (5 вхождений).

    2). Теперь возникает проблема: index нормально отображается в браузере, но не отображается в приложении. Это возникает из-за
    <script type="module" crossorigin> в коде index.html. Чтобы это исправить, сборку надо запускать вместо обычного npm run build
